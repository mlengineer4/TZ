# Диаграмма прецедентов: Библиотечная система

## Описание предметной области

Библиотечная система предназначена для автоматизации работы библиотеки. Она позволяет читателям искать и бронировать книги, а библиотекарям — выдавать книги, принимать возврат и управлять библиотечным фондом.

## Акторы

- **Читатель** — пользователь библиотеки, который ищет, бронирует, получает и возвращает книги.
- **Библиотекарь** — сотрудник библиотеки, который выдаёт книги, принимает возврат и управляет фондом.
- **Система уведомлений** — внешняя система, которая отправляет уведомления читателю.

## Прецеденты

- **Искать книги** — поиск нужной книги в каталоге.
- **Бронировать книгу** — оформление бронирования книги читателем.
- **Получать книгу** — получение книги читателем у библиотекаря.
- **Возвращать книгу** — возврат книги в библиотеку.
- **Управлять фондом** — добавление, удаление и изменение информации о книгах.
- **Проверять наличие книги** — проверка, доступна ли книга.
- **Отправлять уведомление** — отправка сообщения о бронировании.

## Отношения

- Читатель связан с поиском, бронированием, получением и возвратом книги.
- Библиотекарь связан с получением книги, возвратом книги и управлением фондом.
- Система уведомлений связана с отправкой уведомлений.
- Отношение `include` используется между «Бронировать книгу» и «Проверять наличие книги», потому что при бронировании наличие книги проверяется обязательно.
- Отношение `include` используется между «Получать книгу» и «Проверять наличие книги», потому что перед выдачей нужно проверить наличие книги.
- Отношение `extend` используется между «Отправлять уведомление» и «Бронировать книгу», потому что уведомление является дополнительным действием при бронировании.

## Код PlantUML

Код диаграммы находится в файле `usecase_ФИО.puml`.

## Ответы на контрольные вопросы

### 1. Что такое диаграмма прецедентов и для чего она используется?

Диаграмма прецедентов — это UML-диаграмма, которая показывает, какие функции предоставляет система и кто ими пользуется. Она используется для описания функциональных требований к системе.

### 2. Чем актор отличается от прецедента?

Актор — это внешний пользователь, роль или другая система, которая взаимодействует с системой.

Прецедент — это функция или действие, которое система предоставляет актору.

### 3. Что обозначает системная граница?

Системная граница показывает, какие функции находятся внутри системы, а какие участники находятся снаружи. Обычно она изображается прямоугольником с названием системы.

В этой работе системная граница называется «Библиотечная система».

### 4. В чём разница между отношениями include и extend?

`include` означает обязательное включение одного прецедента в другой.

Например, «Бронировать книгу» обязательно включает «Проверять наличие книги».

`extend` означает дополнительное поведение, которое выполняется только при определённых условиях.

Например, «Отправлять уведомление» является дополнительным действием при бронировании книги.

### 5. Почему Mermaid не подходит для построения полноценных use case диаграмм?

Mermaid не подходит для построения полноценных use case диаграмм, потому что он не поддерживает стандартный UML-синтаксис для акторов, прецедентов и связей `include` и `extend`.

Для полноценной диаграммы прецедентов лучше использовать PlantUML.

### 6. Какой синтаксис PlantUML используется для определения актора и прецедента?

Для определения актора используется команда `actor`.

Пример: `actor "Читатель" as Reader`.

Для определения прецедента используется команда `usecase`.

Пример: `usecase "Искать книги" as SearchBooks`.

### 7. Как в PlantUML задать отношение include?

Отношение `include` задаётся пунктирной стрелкой с подписью `<<include>>`.

Пример: `ReserveBook ..> CheckAvailability : <<include>>`.

Это означает, что бронирование книги обязательно включает проверку наличия книги.

### 8. Какие правила именования акторов и прецедентов вы знаете?

Акторы называются существительными, которые обозначают роль или внешнюю систему.

Примеры акторов: «Читатель», «Библиотекарь», «Система уведомлений».

Прецеденты называются глаголом с существительным, потому что они обозначают действия системы.

Примеры прецедентов: «Искать книги», «Бронировать книгу», «Управлять фондом».
