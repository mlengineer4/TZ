## Описание

**Task Manager** — это учебный проект для управления задачами.  
Приложение позволяет создавать задачи, отмечать их выполнение и хранить список дел.

Проект помогает организовать работу, учебу и личные планы.

***Главная идея проекта*** — сделать простой и удобный инструмент для контроля задач.

---

## Цель проекта

Цель проекта — создать приложение, которое позволяет пользователю быстро управлять задачами.

> Хорошая система задач помогает не забывать важные дела и правильно распределять время.

## Пример кода

```javascript
const tasks = [];

function addTask(title) {
    const task = {
        id: Date.now(),
        title: title,
        completed: false
    };

    tasks.push(task);
}

addTask("Сделать практическую работу");
console.log(tasks);
```

В проекте используется функция `addTask()` для добавления новой задачи.

## Ответы на контрольные вопросы

### 1. Чем отличается вложенная цитата от обычной?

Обычная цитата создаётся с помощью одного символа `>`.

```md
> Обычная цитата
```

Вложенная цитата создаётся с помощью нескольких символов `>`.

```md
> Первый уровень цитаты
>> Второй уровень цитаты
```

Вложенная цитата находится внутри другой цитаты.

### 2. Как сделать сноску в Markdown и где она отображается?

Сноска создаётся с помощью обозначения `[^ключ]` в тексте и пояснения внизу документа.

Пример:

```md
Текст со сноской[^1].

[^1]: Пояснение к сноске.
```

Сноска обычно отображается внизу документа или раздела.

### 3. В чём разница между - [ ] и - [x] в списках задач?

`- [ ]` означает невыполненную задачу.  
`- [x]` означает выполненную задачу.

Пример:

```md
- [x] Сделано
- [ ] Не сделано
```

### 4. Какие два способа вставки эмодзи вы знаете?

Первый способ — вставить эмодзи напрямую, например: ✅ 🚀 ⚠️.  
Второй способ — использовать код эмодзи, например `:smile:` или `:rocket:`.

### 5. Зачем нужны HTML-вставки? Приведите пример.

HTML-вставки нужны, чтобы добавить в Markdown элементы, которых нет в обычном синтаксисе Markdown.

Пример:

```html
<details>
<summary>Подробнее</summary>
Дополнительная информация.
</details>
```

Такой блок можно раскрывать и скрывать.

### 6. Почему важно использовать правильные имена для внутренних ссылок?

Правильные имена нужны, чтобы внутренние ссылки переходили к нужным разделам документа.  
Если ссылка указана неправильно, переход по ней работать не будет.

Пример:

```md
[Перейти к описанию](#описание)
```

### 7. Что произойдёт, если в таблице окажется разное количество столбцов в разных строках?

Таблица может отобразиться некорректно. Некоторые строки могут «съехать», а Markdown-редактор может не распознать таблицу правильно.
